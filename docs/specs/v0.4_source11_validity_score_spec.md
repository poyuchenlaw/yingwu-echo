## 1. Purpose

`validity_score` is a single float in `[0.0, 1.0]` that quantifies how much **genuine emotional weight** a piece of writing carries. It is **not** a quality rating for prose style вҖ” it is a signal-quality gate for the soul-analysis pipeline. Low-validity writing produces noisy wuxing/celestial assignments; high-validity writing produces stable, meaningful ones.

The score feeds directly into:
- Wuxing assignment confidence
- Monster rarity tier determination
- Whether a writing triggers a Resonance Altar event

**Banned framing:** Do not describe this score using the words зҷӮзҷ’/жІ»зҷӮ/и§ЈжҶӮ/зҙ“еЈ“/иЁәж–· or their English equivalents in any user-facing copy.

---

## 2. Formula

```
validity_score = clamp(
    0.3 Г— length_coef
  + 0.2 Г— repeat_coef
  + 0.5 Г— llm_genuineness,
  0.0,
  1.0
)
```

Where `clamp(x, 0.0, 1.0) = max(0.0, min(1.0, x))`.

---

## 3. Component Definitions

### 3.1 `length_coef` вҖ” Length Coefficient

```
char_count = len(content)   # Unicode codepoints, not bytes
length_coef = clamp(log(char_count / 100.0) / log(10), 0.0, 1.0)
```

- Uses **log base 10** of the ratio `char_count / 100`.
- The denominator 100 is the "baseline" character count (вүҲ 2вҖ“3 short sentences).
- `char_count = 100` вҶ’ `length_coef = 0.0` (just at baseline, no bonus)
- `char_count = 1000` вҶ’ `length_coef = 1.0` (ceiling reached at 1000 chars)
- `char_count < 100` вҶ’ clamped to `0.0`
- `char_count = 300` вҶ’ `log10(3) вүҲ 0.477`

Implementation note: if `char_count == 0`, return `length_coef = 0.0` (no divide-by-zero guard needed because `log(0/100)` is handled by the clamp at 0).

### 3.2 `repeat_coef` вҖ” Repetition Penalty Coefficient

Measures 3-gram (trigram) repetition rate over the content string.

**Algorithm:**
1. Tokenize content into Unicode codepoints (not words).
2. Extract all overlapping trigrams (sliding window of 3 codepoints).
3. Compute `repeat_rate = (duplicate_trigram_count) / max(total_trigrams, 1)` where `duplicate_trigram_count` = total occurrences of any trigram that appears more than once minus one per trigram (i.e., total trigrams minus distinct trigrams).

```
total_trigrams  = max(len(content) - 2, 0)
distinct_trigrams = len(set(all_trigrams))
repeat_rate = (total_trigrams - distinct_trigrams) / max(total_trigrams, 1)
repeat_coef = 1.0 - (2.0 Г— repeat_rate) if repeat_rate > 0.30 else 1.0
repeat_coef = clamp(repeat_coef, 0.0, 1.0)
```

- If `repeat_rate вүӨ 0.30`: `repeat_coef = 1.0` (no penalty)
- If `repeat_rate > 0.30`: apply a `Г—0.5` effective penalty via the formula above (e.g., `repeat_rate = 0.50` вҶ’ `repeat_coef = 0.0`)
- The threshold of 30% matches empirical observation: natural prose rarely exceeds 25% trigram repetition; spam/padding typically exceeds 40%.

### 3.3 `llm_genuineness` вҖ” LLM Self-Evaluation

The LLM is instructed inside every prompt (via the `promptSuffix` constant in `prompts/templates.go`) to return `validity_score` as part of its JSON response, using this rubric:

| Range | Interpretation |
|-------|----------------|
| 0.8вҖ“1.0 | Sincere, specific, emotionally weighted; вүҘ120 Unicode characters |
| 0.5вҖ“0.79 | Short but honest, OR longer but surface-level |
| 0.2вҖ“0.49 | Perfunctory, repetitive, low emotional content |
| 0.0вҖ“0.19 | Padding, meaningless repetition, or completely off-topic |

The LLM's returned `validity_score` JSON field is used directly as `llm_genuineness`.

**Constraint:** `llm_genuineness` is clamped to `[0.0, 1.0]` by `ParseAnalysis` before use in the formula.

## 5. Boundary Case Examples

### Case A вҖ” High-Quality Long Writing (expected score вүҲ 0.88)

```
content = "д»ҠеӨ©й–ӢдәҶдә”еҖӢе°ҸжҷӮзҡ„жңғпјҢе…ЁзЁӢжІ’жңүж„Ҹзҫ©пјҢжүҖжңүдәәйғҪеңЁиӘӘе»ўи©ұ..." (180 chars)
```

- `char_count = 180` вҶ’ `length_coef = log10(1.8) вүҲ 0.255`
- `repeat_rate вүҲ 0.12` (natural prose) вҶ’ `repeat_coef = 1.0`
- `llm_genuineness = 0.90` (specific, emotionally charged)
- `validity_score = 0.3Г—0.255 + 0.2Г—1.0 + 0.5Г—0.90 = 0.077 + 0.20 + 0.45 = 0.727`

> Note: `length_coef` at 180 chars is relatively low. The 0.5 weight on `llm_genuineness` dominates. A 400-char piece with the same quality would score вүҲ 0.86.

### Case B вҖ” Short Honest Writing (expected score вүҲ 0.42)

```
content = "зқЎеүҚжғіиӘӘиӘӘд»ҠеӨ©гҖӮеҫҲзҙҜпјҢдҪҶйӮ„еҘҪгҖӮжҳҺеӨ©з№јзәҢеҠ жІ№гҖӮ" (23 chars)
```

- `char_count = 23` вҶ’ `log10(0.23) < 0` вҶ’ `length_coef = 0.0`
- `repeat_rate вүҲ 0.05` вҶ’ `repeat_coef = 1.0`
- `llm_genuineness = 0.45` (short but sincere)
- `validity_score = 0.0 + 0.2Г—1.0 + 0.5Г—0.45 = 0.0 + 0.20 + 0.225 = 0.425`

### Case C вҖ” Padding / Repetition Spam (expected score вүҲ 0.04)

```
content = "зҙҜзҙҜзҙҜпјҢзңҹзҡ„еҘҪзҙҜпјҢзҙҜзҙҜзҙҜпјҢзңҹзҡ„еҘҪзҙҜпјҢзҙҜзҙҜзҙҜпјҢзңҹзҡ„еҘҪзҙҜпјҢзҙҜзҙҜзҙҜгҖӮ" (35 chars)
```

- `char_count = 35` вҶ’ `length_coef = 0.0`
- Trigrams heavily repeated: `repeat_rate вүҲ 0.72` вҶ’ `repeat_coef = max(0, 1 - 2Г—0.72) = max(0, -0.44) = 0.0`
- `llm_genuineness = 0.10` (padding)
- `validity_score = 0.0 + 0.0 + 0.5Г—0.10 = 0.05`

### Case D вҖ” Long Padding (expected score вүҲ 0.10)

```
content = "еҗғйЈҜеҗғйЈҜеҗғйЈҜеҗғйЈҜпјҢеҗғйЈҜеҗғйЈҜеҗғйЈҜеҗғйЈҜпјҢд»ҠеӨ©еҗғйЈҜпјҢжІ’д»ҖйәјпјҢе°ұжҳҜеҗғйЈҜпјҢеҗғйЈҜиЁҳйҢ„пјҢеҗғйЈҜеҗғйЈҜпјҢе®ҢгҖӮ" (49 chars, high repeat)
```

- `char_count = 49` вҶ’ `length_coef = 0.0`
- `repeat_rate вүҲ 0.65` вҶ’ `repeat_coef = 0.0`
- `llm_genuineness = 0.05` (near-total padding)
- `validity_score = 0.0 + 0.0 + 0.5Г—0.05 = 0.025` вҶ’ effective вүҲ 0.03

### Case E вҖ” Very Long Genuine Writing (expected score вүҲ 0.93)

```
content = "дёҖеҖӢдәәеңЁжҲҝй–“иЈЎ..." (350 chars, emotionally specific, low repetition)
```

- `char_count = 350` вҶ’ `length_coef = log10(3.5) вүҲ 0.544`
- `repeat_rate вүҲ 0.08` вҶ’ `repeat_coef = 1.0`
- `llm_genuineness = 0.92`
- `validity_score = 0.3Г—0.544 + 0.2Г—1.0 + 0.5Г—0.92 = 0.163 + 0.20 + 0.46 = 0.823`

> At 1000+ chars with the same LLM score: `validity_score = 0.3Г—1.0 + 0.20 + 0.46 = 0.96` (clamped to 1.0 if exceeded).

## 7. Rationale for Weights

| Weight | Component | Rationale |
|--------|-----------|-----------|
| 0.50 | `llm_genuineness` | The LLM has access to semantic content; structural signals alone cannot detect emotionally authentic short writing |
| 0.30 | `length_coef` | Longer writing empirically correlates with genuine reflection; short writing is not disqualified but not rewarded |
| 0.20 | `repeat_coef` | Repetition is the clearest objective spam signal, but its absence alone does not prove quality |

The 0.30/0.20/0.50 split was chosen so that:
1. A spammer cannot achieve > 0.5 by writing long repetitive text (repeat_coef collapses)
2. A short genuine reflection (~30 chars, llm=0.8) scores вүҲ 0.40, which is "usable but flagged"
3. A long, genuine, specific piece (350+ chars, llm=0.9) scores вүҲ 0.82+, reaching the high-quality tier
